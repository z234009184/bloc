作者: CodeMan 日期: 五月 20, 2017  23:00:40 分类: iOS

Mach-O 介绍

Mach-O 其实是 Mach Object 文件格式的缩写, 属于Mach-O格式的常见文件有

  • 目标文件.o
  • 库文件
    • .a 静态库其实就是多个.o链接到一起
    • .dylib
    • framework
  • 可执行文件
  • dyld (动态链接器)
  • .dsym( 符号表)

Mach-O 的基本结构

  • Header
    • 文件类型、目标架构类型等
  • Load commands
    • 描述文件在虚拟内存中的逻辑结构、布局
  • Raw segment data
    • 在Load commands 中定义的 Segment 原始数据

如图所示:

查看 Mach-O 文件结构

  • 命令行工具

    • file 查看文件类型
    • otool 用来查看特定部分和段的内容
    • lipo 用于多架构文件的处理
      • lipo -info <文件路径>
      • lipo <文件路径> -thin <架构类型> -output <导出的文件目录>
      • lipo <文件路径1> <文件路径2> -output <输出的文件路径>

  • GUI 工具

    • MachOView

dyld 用于加载以下几种Mach-O格式文件:

  1. 可执行文件
  2. dylib 动态库
  3. bundle

App从开发到安装到手机上的过程

  1. 编写代码
  2. 编译、链接、签名 生成 .app 包
  3. .app包内包含可执行文件和一些资源
  4. 压缩.app包为ipa文件
  5. ipa文件可以上传到 AppStore 或者 其他商店
  6. 用户从AppStore 或其他商店 安装ipa

逆向App思路

  1. 界面分析
    • Cycript、Reveal
  2. 代码分析
    • 对Mach-O文件静态分析
    • 主要工具有: MachOView、class-dump、Hopper Disassembler、ida
  3. 动态调试
    • debugserver、LLDB
  4. 编写代码注入到App中
    • theos hook方式编写deb安装包 Cydia安装deb插件(已越狱)
    • 重签名、打包ipa(未越狱)
